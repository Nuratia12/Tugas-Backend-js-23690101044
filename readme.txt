1. Perbedaan tanggung jawab Front-End dan Back-End
(Analogi toko dalam pembuatan web)
Front-End
Seperti tampilan toko (etalase, rak, kasir).
Bertugas mengatur:
Tampilan website (warna, tombol, form)
Interaksi pengguna (klik, isi form)
Kenyamanan pengguna saat memakai web
Back-End
Seperti bagian belakang toko (gudang, pembukuan, sistem stok).
Bertugas:
Mengolah data dari pengguna
Menyimpan dan mengambil data dari database
Menjaga keamanan dan logika sistem
 Kesimpulan:
Front-End = yang dilihat dan digunakan
Back-End = yang bekerja di balik layar
2. Perbedaan GET dan POST dalam pengiriman data form
(Analogi toko online)
GET
Seperti bertanya harga barang secara terbuka.
Data terlihat di URL
Cocok untuk mengambil data
Tidak aman untuk data penting
POST
Seperti menyerahkan formulir pesanan ke kasir.
Data tidak terlihat
Lebih aman
Cocok untuk mengirim data penting
Kapan menggunakan POST?
Gunakan POST saat:
Login dan registrasi
Mengirim password atau data pribadi
Menyimpan data ke database
3. Mengapa validasi input harus di Back-End walau sudah ada di Front-End?
Karena:
Front-End bisa dimanipulasi
Seperti pelanggan masuk lewat pintu belakang toko.
Keamanan sistem
Back-End mencegah data palsu dan serangan berbahaya.
Back-End adalah pengambil keputusan akhir
Seperti pemilik toko yang menentukan transaksi sah atau tidak.
 Analogi:
Front-End = satpam
Back-End = brankas
Kalau satpam lengah, brankas tetap harus aman
